Help & Data Format
JSON schema
Import expects a JSON array of expense objects. required fields must be present. optional fields can be omitted.
FieldTypeValuesNotes
name requiredstringanyDescription of the purchase
amt requirednumber250Dollar amount — plain number, no $ sign. Use negative for a refund/return.
month requirednumber1–12Month number
year requirednumber20264-digit year. Defaults to planner start year if omitted.
priority requiredstring"committed" "optional" "nicetohave"committed=must buy · optional=likely · nicetohave=want but could skip
purchased optionalbooleantrueInclude only if already bought. Omit otherwise.
note optionalstringanyAny extra context. Omit if empty.
id optionalstring/numberany unique valueAuto-generated if omitted. Used by Merge to detect duplicates.

Workflow tips
• Use ⬆ Export to copy your current data as JSON — handy for backups or AI bulk edits.
• Use ⬇ Import → Merge to add new items without losing existing data.
• Use ⬇ Import → Replace when an AI has rewritten your full list.
• The status button cycles: Committed → Optional → Nice to Have → Purchased. For refunds (negative amounts): Expected → Possible → Hopeful → Credited.
• Click the pencil icon to rename any item inline.
